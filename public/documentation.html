Movie API Documentation

This API can be used to query movies, actors, and genres. More specifically, this API can return data about movie descriptions, genres, directors, image URLs. It can allow new users to register, update thier info, add movies to their favorites, remove movies from their favorites, and allow them to deregister.

Get All Movies

Return a list of all movies to the user.

  • Endpoint: /movies
  • Method Type: GET
  • Query Parameters: none
  • Request body content: none
  • Response format: JSON objects of all movies, each of which contain the movie's title, description, director, genre, image url, and featured status.

Get One Movie

Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user.

  • Endpoint: /movies/:Name
  • Method Type: GET
  • Query Parameters: Name of the movie.
  • Request body content: none
  • Response format: JSON object containing the movie's title, description, director, genre, image url, and featured status.
  • Response example: {
    "title": "Ex Machina",
    "description": "A young programmer is selected to participate in a ground-breaking experiment in synthetic intelligence by evaluating the human qualities of a highly advanced humanoid A.I.",
    "director": {
    "name": "Alex Garland"
    },
    "genre": {
    "name": "Drama"
    },
    "imageurl": "https://m.media-amazon.com/images/M/MV5BMTUxNzc0OTIxMV5BMl5BanBnXkFtZTgwNDI3NzU2NDE@._V1_SY1000_CR0,0,674,1000_AL_.jpg",
    "featured": false
    }

Get a Genre

Return data about a genre (description) by name/title (e.g., “Thriller”).

  • Endpoint: /genres/:Name
  • Method Type: GET
  • Query Parameters: Name of the genre.
  • Request body content: none
  • Response format: JSON Object of the genre's name and description
  • Response example: {
    "name": "Thriller",
    "description": "Thriller film, also known as suspense film or suspense thriller, is a broad film genre that involves excitement and suspense in the audience."
    }

Get a Director

Return data about a director (bio, birth year, death year) by name.

  • Endpoint: /directors/:Name
  • Method Type: GET
  • Query Parameters: Name of the director.
  • Request body content: none
  • Response format: JSON Object of the director's name, bio, and birth date.
  • Response example: {
    "name": "Quentin Tarantino",
    "bio": "Quentin Jerome Tarantino was born in Knoxville, Tennessee. His father, Tony Tarantino, is an Italian-American actor and musician from New York, and his mother, Connie (McHugh), is a nurse from Tennessee.",
    "birthdate": "1963-03-27T00:00:00.000Z"
    }

Create a New User

Allow new users to register with a name, username, password, email, and date of birth.

  • Endpoint: /users/new
  • Method Type: POST
  • Query Parameters: none
  • Request body content: JSON
  • Request example: { "name" : "Evan Rogers", "username" : "evan123", "password" : "rogers123", "email" : "evan@gmail.com", "birth_date" : "02-06-1991", "favorite_movies" : "5de85a49334613d628eb855e" }
  • Response format: JSON Object with the new user's info containing their ID, name, username, password, email, date of birth, and favorite movies.
  • Respone example: {
    "_id": "5df02c6ac11858b16b7ba0dc",
    "name": "Evan Rogers",
    "username": "evan123",
    "password": "rogers123",
    "email": "evan@gmail.com",
    "birth_date": "1991-02-06T00:00:00.000Z",
    "favorite_movies": [
    {
    "title": "Fight Club"
    }
    ]
    }

Update a User

Allow users to update their user info (username, password, email, date of birth).

  • Endpoint: /user/:Username
  • Method Type: PUT
  • Query Parameters: Username of the user.
  • Request body content: JSON
  • Request example: { "name" : "Evan Rogers", "username" : "evan123", "password" : "rogers123", "email" : "evanrogers@gmail.com", "birth_date" : "02-06-1991"}
  • Response format: JSON Object with the updated info contain the user's ID, name, username, password, email, date of birth, and favorite movies.
  • Respone example: {
    "_id": "5df02c6ac11858b16b7ba0dc",
    "name": "Evan Rogers",
    "username": "evan123",
    "password": "rogers123",
    "email": "evanrogers@gmail.com",
    "birth_date": "1991-02-06T00:00:00.000Z",
    "favorite_movies": [
    {
    "title": "Fight Club"
    }
    ]
    }

Add Movie to Favorites

Allow users to add a movie to their list of favorites.

  • Endpoint: /users/:Username/Movies/:MovieID
  • Method Type: POST
  • Query Parameters: Username of the user and movie ID of the movie.
  • Request body content: none
  • Response format: JSON Object contain the user's ID, name, username, password, email, date of birth, and new favorite movies.
  • Response example: {
    "_id": "5df0293d3883cab12e9c56e2",
    "name": "Evan Rogers",
    "username": "evan123",
    "password": "rogers123",
    "email": "evanrogers@gmail.com",
    "birth_date": "1991-02-06T00:00:00.000Z",
    "favorite_movies": [
    {
    "title": "Fight Club"
    },
    {
    "title": "Inception"
    }
    ]
    }

Delete Movie from Favorites

Allow users to remove a movie from their list of favorites.

  • Endpoint: /users/:Username/Movies/:MovieID
  • Method Type: DELETE
  • Query Parameters: Username of the user and movie ID of the movie.
  • Request body content: none
  • Response format: Message indicating the movie was removed from favorites.

Delete a User

Allow existing users to deregister.

  • Endpoint: /users/:Username
  • Method Type: DELETE
  • Query Parameters: Username of the user.
  • Request body content: none
  • Response format: Message indicating that the user has been deregistered.
